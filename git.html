Creating gh-pages branch

This is what I did to create a gh-pages branch in a local working repository.

  • Forked the project dcm4che/dcm4che to make a project bradleyross/dcm4che. I then added the branch gh-pages. At this point, the contents of the gh-pages and master branches are the same.
  • Created a directory on my workstation for the gh-pages branch and then used the cd command to set that as the current directory.
  • Issued the command git clone https://www.github.com/bradleyross/dcm4che . This created a clone of the master branch in the directory.
  • Used the command git checkout gh-pages. This changed the contents of the directory to the gh-pages branch.
  • Created a subdirectory in the directory named ToBeDeleted and moved everything but .gitignore to this directory. I also added .DS_Store as a line in the .gitignore file. (This is for Apple MacOS workstations but won't interfere with other systems.)
  • I then carried out the following commands.
    • rm -r ToBeDeleted
    • git add .gitignore
    • git commit . -m "Removing all non-documentation files
    • git push origin gh-pages
